Mini Library App
معرفی پروژه

این پروژه مربوط به دستور کار آزمایشگاه درس مبانی فناوری اطلاعات دانشکده برق و کامپیوتر دانشگاه تهران است.

هدف پروژه، پیاده‌سازی یک برنامه‌ی ساده‌ی خط فرمان (CLI) برای مدیریت کتاب‌های یک کتابخانه کوچک و آشنایی عملی با مفاهیم توسعه نرم‌افزار گروهی و استفاده از Git و GitHub می‌باشد.

در این برنامه امکان افزودن، مشاهده، جستجو و حذف کتاب‌ها فراهم شده و اطلاعات به‌صورت پایدار در فایل ذخیره می‌شوند.
قابلیت‌های برنامه

    افزودن کتاب جدید (عنوان، نویسنده، سال انتشار)
    نمایش لیست تمام کتاب‌ها
    جستجوی کتاب بر اساس عنوان یا نویسنده
    حذف کتاب از کتابخانه
    ذخیره‌سازی اطلاعات به‌صورت پایدار در فایل JSON
    رابط کاربری خط فرمان (CLI)

پیش‌نیازها

    Python نسخه 3.8 یا بالاتر
    Git (برای clone کردن مخزن)

    این پروژه کتابخانه‌ی خارجی خاصی نیاز ندارد.

نحوه دریافت و اجرای برنامه
1. دریافت پروژه

ابتدا مخزن پروژه را clone کنید:

        

bash

git clone https://github.com/nimakhajri/mini-library-app.git

سپس وارد پوشه پروژه شوید:

        

bash

cd mini-library-app

2. اجرای برنامه

برای اجرای برنامه از دستور زیر استفاده کنید:

        

bash

python main.py

پس از اجرا، منوی برنامه در ترمینال نمایش داده می‌شود و می‌توانید از گزینه‌های مختلف استفاده کنید.
ساختار پروژه

        

text

mini-library-app/
│
├── main.py
├── README.md
├── requirements.txt
├── .gitignore
│
├── data/
│   └── books.json
│
└── library/
    ├── book.py
    ├── library_manager.py
    └── storage.py

توضیح پوشه‌ها و فایل‌ها

    main.py: نقطه شروع برنامه و رابط خط فرمان
    library/: شامل منطق اصلی برنامه
        book.py: تعریف کلاس Book
        library_manager.py: مدیریت عملیات کتاب‌ها
        storage.py: مدیریت ذخیره و بازیابی اطلاعات از فایل JSON
    data/books.json: محل ذخیره دائمی اطلاعات کتاب‌ها
    requirements.txt: لیست وابستگی‌ها (در این پروژه خالی است)
    .gitignore: فایل‌ها و پوشه‌های غیرضروری برای Git

نحوه کار با برنامه

پس از اجرای main.py، منویی مشابه زیر نمایش داده می‌شود:

    افزودن کتاب
    نمایش لیست کتاب‌ها
    جستجوی کتاب
    حذف کتاب
    خروج

کاربر با وارد کردن شماره هر گزینه می‌تواند عملیات مورد نظر را انجام دهد.
روند توسعه و Git Workflow

در این پروژه، توسعه نرم‌افزار طبق اصول Git و GitHub انجام شده است:

    استفاده از مخزن عمومی (Public Repository)
    توسعه امکانات در شاخه‌های جداگانه (Feature Branches)
    ثبت تغییرات با Commitهای معنادار
    استفاده از Pull Request برای ادغام تغییرات
    استفاده از Issues برای مدیریت تسک‌ها
